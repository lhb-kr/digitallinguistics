Shakespeare Character Networks Explorer

Quick Guide

What The Network Shows

Each graph is a character network. A node is a character, and an edge means two characters are connected within the selected dramatic unit, such as a scene, act, or full play. Thicker edges usually mean the connection appears more often.

The graph does not prove friendship, influence, or direct address by itself. It shows patterns of dramatic co-occurrence that can help you ask literary questions about scene structure, dialogue, centrality, isolation, and groups of characters.

Top Bar

View Buttons

Switches between the two main ways to use the explorer.

  • Single: shows one play in detail, with timeline controls, character focus, full metrics, and exports for one network.
  • Compare: shows four full-play networks at once for comparing overall network shape across plays.
Play Menus

Selects which play or plays are shown.

  • Single Play Menu: selects the play used in Single view.
  • Compare Play Menus: select the four plays shown in Compare view. Compare view uses full-play networks only.

Single View: Left Panel

Network

Edge Type

Chooses what kind of co-occurrence creates an edge between two characters.

  • Stage: connects characters who appear in the same scene, either because they speak or because stage directions mark them as present. Use this for shared dramatic space, including silent presences.
  • Speech: connects characters when both have spoken lines in the same scene. Use this for dialogue participation. It does not necessarily mean one character speaks directly to the other.
Mode

Changes the time scale of the graph.

  • Full play: shows the network for the entire play.
  • Cumulative act: builds the network from the beginning of the play through the selected act.
  • Act: shows only the selected act.
  • Cumulative scene: builds the network from the beginning of the play through the selected scene.
  • Scene: shows only the selected scene.
  • 3-scene window / 5-scene window: shows a rolling window of recent scenes.

Timeline

Previous, Next, Play

Moves through the selected time mode.

  • Previous: moves to the previous act, scene, cumulative step, or scene window.
  • Next: moves to the next act, scene, cumulative step, or scene window.
  • Play: automatically advances through the available time positions.

In Full play mode, the timeline does not move because the graph is one complete-play snapshot.

Slider
Jumps to a specific act, scene, or window position.

Character

Focus
Selects one character. The selected character and connected characters are emphasized in the graph and described in the right panel.
Clear
Removes the selected character focus.

Encoding And Filters

Node Size

Chooses what makes a character look larger.

  • Weighted degree: the default; larger nodes have stronger or more repeated ties.
  • Degree: larger nodes are connected to more characters.
  • PageRank: larger nodes are connected to characters who are themselves important in the network.
  • Betweenness: larger nodes often sit between otherwise separate parts of the network.
  • Closeness: larger nodes are, on average, closer to all other characters.
  • Eigenvector: larger nodes are connected to well-connected characters.
  • Uniform: all visible characters use the same node size.
Node Color

Chooses what node colors mean.

  • Community: colors characters by detected network groups.
  • Gender: colors characters by available gender metadata.
  • Node metric: colors characters by the selected centrality value.
Community Method

Chooses the algorithm used to find communities: groups of characters more strongly connected to each other than to the rest of the network.

  • Leiden: the default community detection method.
  • Louvain: an alternative method that can be useful for checking whether group patterns are stable.

Community results can change when edge type, time mode, or filters change.

Edge Width

Chooses what makes a tie look thicker.

  • Weight: the default; thicker ties mean more repeated co-occurrence.
  • Uniform: all visible ties use the same thickness.
  • Endpoint degree: thicker ties connect characters with more visible connections.
  • Endpoint weighted degree: thicker ties connect characters with stronger or more repeated ties overall.
  • Endpoint betweenness: thicker ties connect characters with higher betweenness values.
Top Nodes Percentile
Keeps only the top percentage of characters according to the current Node Size metric. For example, 25% keeps the highest quarter of characters for each current graph.
Minimum Degree
Hides characters with fewer visible connections. This is useful for simplifying a dense network.
Minimum Edge Weight
Hides weaker ties. Raising it keeps only connections that occur more often.
Labels
Turns character labels on or off.
Group Characters
Shows or hides collective roles such as attendants, soldiers, players, messengers, or citizens. Hiding them often makes the graph easier to read when focusing on named characters.
Reset Filters
Returns filtering controls to their default values.

Single View: Right Panel

Metrics

Summarizes the currently visible network.

  • Nodes: the number of visible characters.
  • Edges: the number of visible ties between characters.
  • Density: how connected the graph is relative to all possible ties.
  • Q Modularity: how strongly the detected communities are separated.
  • Communities: the number of detected groups in the visible graph.
  • Avg Weight: the average weight of visible ties.
Top Nodes
Lists the highest-ranked visible characters using the currently selected Node Size metric.
Focus

Describes the selected character.

  • Active status: shows whether the character appears in the current view.
  • Centrality values: shows measures such as degree, weighted degree, PageRank, and ego size.
  • Strongest ties: lists the character's strongest visible connections.
  • New and lost ties: shows changes when timeline modes are used.
Export

Saves the current single-play network.

  • JSON: saves network data and current settings.
  • CSV: saves the visible edge list.
  • PNG: saves an image of the current network canvas.

Compare View

Four Network Canvases
Shows four selected full-play networks for broad comparison.
Shared Encoding
The left-panel encoding and filter controls apply to all four networks.
Compare Metrics

Shows numeric metrics for the four selected plays.

  • All Shakespeare Plays Avg: gives the dataset-wide baseline across all plays.
  • Selected plays: lists one metric row for each selected play.

The table includes nodes, edges, density, communities, Q modularity, and average weight.

Compare Export

Saves comparison outputs.

  • JSON: saves comparison data and current settings.
  • CSV: saves the comparison metric table.
  • PNG: saves the four-panel network image.
