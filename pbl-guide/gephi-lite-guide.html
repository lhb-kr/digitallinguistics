Using Gephi Lite to Read Character Networks

This guide teaches the practical order of work: bring in a network, calculate useful metrics, arrange the graph, simplify it carefully, and explain what the visualization suggests.

Open Gephi Lite Find DraCor Data
Example network nodes, edges, communities

Recommended Working Order

Follow this order when you are working independently. You can loop back when the graph changes.

01Load dataDraCor or a saved file
02InspectCheck nodes and edges
03Compute metricsDegree, Louvain, betweenness
04Style graphSize, color, labels
05Run layoutUse ForceAtlas2
06FilterHide minor nodes or weak edges
07RecomputeIf the visible graph changed
08Export or savePNG, GEXF, or JSON
09InterpretExplain the pattern
If you filter or delete nodes before analysis, compute the metrics again. Gephi Lite computes metrics on the graph you are currently working with, so the results can change after filtering.

Load Data

Path A: From DraCor

  1. Go to DraCor.
  2. Choose a corpus.
  3. Open a play or work page.
  4. Open the Tools area.
  5. Click Gephi Lite.

Path B: Open a Saved File

Use this when you continue earlier work.

.json workspace .gexf graph .graphml graph

A Gephi Lite JSON workspace is best for continuing a project because it keeps more of the working state. GEXF is useful for exchanging graph data with desktop Gephi.

Side Panels

Use Appearance, Metrics, Layout, and Filters to transform the graph from raw data into an interpretable visualization.

Appearance

Appearance does not calculate new information. It uses existing attributes or computed metrics to make patterns visible.

Visual choice Good first choice Why it helps
Node size degree Larger nodes show characters with more connections.
Node color modularityClass Colors show communities found by Louvain community detection.
Labels Show fewer labels first Too many labels can make the graph harder to read.

Layout: ForceAtlas2

Layout changes where nodes appear on the screen. It does not change the underlying data. In Gephi Lite, the main layout to teach first is ForceAtlas2.

Use This Setup First

For most DraCor character networks, start here before experimenting.

  1. Compute Degree and Louvain first.
  2. Set node size by degree.
  3. Open Layout and choose ForceAtlas2.
  4. Turn on Adjust sizes?.
  5. Set Scaling ratio to 30.
  6. Keep Gravity at 1.
  7. Keep Edge weight influence at 1 if edge weight means co-occurrence strength. Use 0 if you are unsure.
  8. Run the layout, then stop it when the graph stabilizes.

If the graph looks wrong, change this

Problem you see Try this setting Why
The graph is too crowded. Increase Scaling ratio to 50 or higher. It spreads nodes farther apart.
Large nodes overlap. Turn on Adjust sizes?. The layout considers node size while positioning nodes.
Communities are hard to see. Turn on linLog mode. It often separates clusters more clearly.

Open the setting explanations

Adjust sizes?
Uses node size during layout. Turn it on after you have sized nodes by a metric such as degree, otherwise large nodes may overlap.
barnesHutOptimize and barnesHutTheta
Barnes-Hut makes repulsion faster on larger graphs. barnesHutTheta controls the approximation. The default 0.5 is a good balance; higher values can be faster but less precise.
Edge weight influence
Controls how strongly edge weights affect attraction. Use 1 when weight means strength or frequency. Use 0 when you want every edge to count equally.
Gravity and Strong gravity mode?
Gravity pulls nodes toward the center. Increase it if components drift away. Strong gravity is more forceful, so use it carefully because it can compress the graph.
linLog mode
LinLog mode changes the force model so communities often become more visually separated. It is useful after a first layout pass when your goal is to compare clusters.
Scaling ratio and Slow down?
Scaling ratio is the main spacing control. Increase it to spread a crowded graph. Slow down reduces movement speed and can help when the layout is unstable.

Metrics

Metrics add new columns to the node or edge data. After computing a metric, use its output in Appearance or Filters. Start with Degree, Louvain community detection, and Betweenness centrality.

Recommended student sequence: compute Degree, compute Louvain community detection, compute Betweenness centrality, then style the graph with size by degree and color by modularityClass.

Node Metrics

Degree

output: degree

What it meansThe number of connections a node has. In a character network, a high-degree character is connected to many other characters.

SettingsType of degree: use Degree for ordinary undirected co-occurrence networks. Use incoming or outgoing degree only for directed networks. Edge weight attribute: choose a numeric weight if you want weighted degree.

How to use itUse degree for node size or to filter minor nodes.

Louvain Community Detection

output: modularityClass

What it meansFinds groups of nodes that are more densely connected to each other than to the rest of the graph.

SettingsFast local moves: keep on. Random walk: keep on for exploration. Resolution: default is 1; increase for more, smaller communities; decrease for fewer, larger communities. Edge weight attribute: choose weight if edge strength should matter.

How to use itUse modularityClass for node color. Treat communities as a clue for interpretation, not as automatic proof of social groups.

Betweenness Centrality

output: betweennessCentrality

What it meansShows which nodes often sit on paths between other nodes. In a character network, a high value can point to a bridge character connecting groups.

SettingsNormalize output?: keep on for easier comparison. Edge weight attribute: for a first analysis, leave blank unless you know how your weights should affect path calculation.

How to use itUse it to discuss characters who connect otherwise separate parts of the network.

PageRank

output: pagerank

What it meansScores nodes by considering not only how many connections they have, but also how important their neighbors are.

SettingsAlpha: keep 0.85. Max iterations: keep 100. Tolerance: keep 1e-6. Use edge weight only when weights should affect importance.

How to use itUse it as an optional comparison with degree. It is especially useful for directed link or influence networks.

HITS

outputs: hubs, authorities

What it meansSeparates nodes into hubs and authorities. A hub points to many important nodes; an authority is pointed to by important hubs.

SettingsNormalize output?: keep on. Max iterations: keep 100. Tolerance: keep the default. Use edge weight only if the directed relation has meaningful weights.

How to use itUse mainly for directed networks. It is usually not the first metric for simple drama co-occurrence networks.

Edge Metrics

Disparity

output: disparity

What it meansHighlights edges that are unusually important relative to the weighted connections around a node.

SettingsChoose an Edge weight attribute. This metric is most useful when the graph has meaningful numeric edge weights.

How to use itUse it to filter or inspect unusually strong relationships.

Simmelian Strength

output: simmelianStrength

What it meansMeasures how strongly an edge is embedded in triangles. If two characters share many mutual connections, their edge can have higher Simmelian strength.

SettingsNo settings in Gephi Lite.

How to use itUse it to identify relationships inside tightly connected groups.

Mixed Metric

Louvain Edges Ambiguity

outputs: coMembershipScore, bridgeNessEdgeScore, ambiguityScore, sourceCommunityId, meanAmbiguityScore

What it meansRuns Louvain many times and checks whether edges consistently stay inside the same community. It helps you see where community boundaries are unstable.

SettingsLouvain runs: default is 50. More runs can be more stable but slower. Resolution: same idea as Louvain; higher gives more communities. Edge weight attribute: select weight only if weighted communities are meaningful.

How to use itUse after you already understand degree, Louvain, and betweenness. It is useful when you want to question whether the detected communities are stable.

Filters and Node Removal

Filtering hides parts of the graph while you explore. Deleting changes the dataset. Start with filters, then delete only when you are sure a node should not be part of the final analysis.

Degree Filter

Hide nodes with very few connections, for example nodes with degree lower than 2.

Edge Weight Filter

Hide weak relationships when the edge weight represents frequency or strength.

Attribute Filter

Focus on a category such as role, group, act, gender, or any attribute available in the data.

Optional: If You Need Raw Data

If DraCor does not provide the exact network you need, you can start from an edge list. Gephi Lite opens graph files such as GEXF or GraphML, so a spreadsheet usually needs to be converted first. 

source,target,weight
Hamlet,Ophelia,5
Hamlet,Claudius,8
Ophelia,Polonius,3

How to read this table

source and target are the two connected nodes. weight is the strength or frequency of the relationship.

Convert the table to a graph format with a tool such as Table2Net or desktop Gephi, then open the graph in Gephi Lite.

Additional Note

Gephi Lite is the browser-based version of Gephi. For larger projects or more advanced functions, students can also use the desktop version of Gephi, which has more tools and export options.
