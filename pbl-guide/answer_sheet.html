110 pts
What is the most distinctive feature of PBL?
The instructor lectures on all content before giving a test
A problem is presented before instruction, and students solve it proactively
Individual students solve problems alone
The instructor provides the correct answer to the problem
Answer: PBL reverses the traditional "teach first, test later" model. A real-world problem is presented before instruction, and students work in teams to solve it proactively. In the PBL comparison table, the starting point is "A real-world problem is presented," contrasting with lecture-based classes where the instructor explains theory first.
210 pts
What is the instructor's role in PBL?
Knowledge transmitter — lectures on all content
Supervisor — controls student behavior
Learning facilitator — supports learning through feedback, guidance, and probing questions
Evaluator — only creates exam questions
Answer: In PBL, the instructor's role shifts from "Knowledge transmitter" (lecture-based) to Learning facilitator. According to the PBL Guide, the facilitator: provides feedback during team discussions, guides discussions toward problem-solving, asks probing questions to grasp the problem's essence, and delivers mini-lectures for wrap-up & synthesis.
310 pts
Which is NOT one of the four IFLA areas in the Task Performance Plan?
Ideas (Hypotheses/Solutions)
Facts (Known Information)
Feedback
Learning Issues (What We Need to Learn)
Answer: Feedback is NOT part of IFLA. The four areas of the Task Performance Plan (과제수행계획서) are: I — Ideas (가설 · 해결안), F — Facts (알고 있는 사실), L — Learning Issues (더 알아야 할 사항), and A — Action Plans (실천 계획). These form the core weekly document in PBL and iterate across rounds.
410 pts
Which represents proper PBL teamwork?
"I'll do Ch.1, you do Ch.2, you do Ch.3. Let's just combine them!"
"Let's each share what we researched, teach each other, and develop a solution together"
"The team leader will organize everything; the rest just present"
"We don't know, so let's just ask the professor for the answer"
Answer: The PBL Guide emphasizes that Team Learning ≠ Task Division. "You do Part 1, I'll do Part 2, let's just combine them" is explicitly called out as NOT PBL teamwork. True PBL teamwork means all members participate in every stage, share what they researched, teach each other, and develop solutions together. A "Team" requires shared tasks, shared responsibility, trust, and commitment — unlike a simple "Group" which is just a collection of individual tasks.
510 pts
Which is NOT a characteristic of PBL problems?
Ill-structured — no single correct answer, multiple solution paths
Authentic — problems that could occur in the real world
Simplicity — easy and straightforward, solvable by anyone immediately
Relevance — problems that learners feel are personally relevant
Answer: Simplicity is NOT a PBL characteristic. The PBL Guide's 4 Core Principles describe PBL problems as: Ill-Structured ("messy" problems with no single correct answer), requiring Student Agency, driven by Collaborative Inquiry, and following an Iterative Cycle. Simple, easy problems do not align with PBL's purpose — the problems must be complex enough to require multiple perspectives and research.
610 pts
What is the correct order of PBL's 5 stages?
Problem presentation → Class orientation → Solution presentation → Follow-up → Evaluation
Class orientation → Problem presentation → Follow-up (iterative) → Solution presentation → Review & evaluation
Evaluation → Problem presentation → Class orientation → Solution presentation → Follow-up
Class orientation → Solution presentation → Problem presentation → Evaluation → Follow-up
Answer: Based on the Barrows & Myers (1993) model, PBL follows a 5-stage flow: Stage 1 Class Orientation (team building, ground rules) → Stage 2 Problem Presentation (real-world problem, IFLA begins) → Stage 3 Follow-up / Iterative (synthesize, revise, repeat over several weeks) → Stage 4 Solution Presentation (teams present completed solutions) → Stage 5 Review & Evaluation (reflection, self/peer evaluation).
710 pts
[True/False] "In PBL, once roles are assigned, each member only needs to complete their own role."
True
False
Answer: False. Role assignment is possible, but completing your own role doesn't end your responsibility. The PBL Guide states that all team members must participate in every stage and discuss together. The IFLA important notes also emphasize: "Content studied individually must be taught to teammates. Studying 'what we need to learn' is a whole-team task."
810 pts
Which activity corresponds to "Learning Issues" in the Task Performance Plan?
Speculate on the causes and effects of the problem
Organize facts you already know
Identify what more you need to know to solve the problem
Create a concrete action plan
Answer: In the IFLA framework, L — Learning Issues (더 알아야 할 사항) is about identifying knowledge gaps. The PBL Guide defines it as: "Things that need to be known or understood to solve the problem." The activity involves identifying knowledge gaps and specifying learning needs. The other options correspond to different IFLA areas — Ideas (speculating causes/effects), Facts (organizing prior knowledge), and Action Plans (concrete action plans).
910 pts
Select ALL research-proven effects of PBL. (Multiple choice)
Improved information retention
Enhanced problem-solving skills
Automatic test score improvement
Improved self-directed learning
Enhanced collaborative learning
Answer: All except "Automatic test score improvement." The PBL Guide's Benefits section lists four research-proven effects: 1) Improved Retention — learning becomes meaningful when learners determine their own needs; 2) Problem-Solving Skills — metacognition and problem-solving efficiency are mutually reinforced; 3) Self-Directed Learning — learners develop self-directed abilities through the problem-solving process; 4) Collaborative Learning — students improve collaborative skills through shared responsibility. "Automatic test score improvement" is NOT a proven PBL effect.
