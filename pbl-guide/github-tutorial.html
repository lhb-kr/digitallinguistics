Create Your Website on GitHub

Reference: You can also check this guide or this visual walkthrough for additional help.
Step 1

Sign Up for a GitHub Account

If you don't have a GitHub account yet, creating one takes just a minute.

  1. Go to github.com.
  2. Click Sign up in the top-right corner.
  3. Enter your email address, create a password, and choose a username.
  4. Complete the CAPTCHA puzzle to prove you are human.
  5. Verify your email by entering the code GitHub sends to your inbox.
Important: Your username becomes part of your website URL. For example, if your username is alex, your site will be alex.github.io. Pick something clean and professional.
Step 2

Create the Repository

GitHub allows one personal website per account. To set it up, you need a repository with a specific name.

  1. Log in to GitHub and click the + icon in the top-right corner, then select New repository.
  2. For the repository name, type username.github.io (replace username with your actual GitHub username).
  3. Set the visibility to Public. GitHub Pages is free for public repositories.
  4. Check the box that says Add a README file.
  5. Click Create repository.
Why this name? GitHub recognizes the username.github.io pattern and automatically treats it as your personal website repository.
Step 3

Add Your index.html File

The index.html file is the front door of your website. This is what visitors see first when they open your URL. You have two options:

Option A

Create a new file

  1. In your repository, click Add file and select Create new file.
  2. In the filename box, type index.html.
  3. Paste the HTML code below into the editor:
index.html <!DOCTYPE html> <html lang="en"> <head> <meta charset="UTF-8"> <title>My Personal Webpage</title> </head> <body> <h1>Hello, World!</h1> <p>Welcome to my new personal site hosted on GitHub.</p> </body> </html>
  1. Scroll down, add a commit message, and click Commit changes.
Option B

Upload your existing file

If you already have an index.html file (for example, one you made for an assignment), you can upload it directly.

  1. In your repository, click Add file and select Upload files.
  2. Drag and drop your index.html file, or click to browse.
  3. Add a commit message and click Commit changes.
Step 4

Turn On GitHub Pages

Naming the repository correctly often activates GitHub Pages automatically, but it is a good idea to double-check the settings.

  1. Click the Settings tab at the top of your repository page.
  2. In the left sidebar, click Pages (under "Code and automation").
  3. Under Build and deployment, confirm these settings:
  • Source Deploy from a branch
  • Branch main and folder set to / (root)
  1. Click Save.
Step 5

See Your Live Website

At the top of the Pages settings screen, you will see a message that says your site is live, along with a link like:

https://username.github.io/

Click that link and you should see your page.

Don't panic: It can take 1 to 5 minutes for the site to go live the very first time. If you see a 404 error, wait a moment and refresh the page.
