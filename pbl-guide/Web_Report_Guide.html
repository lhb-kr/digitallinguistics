Building Your Interactive Web Page Report

Turn your project report into a single self-contained HTML page using an AI chatbot, then publish it on GitHub.

1 What You Will Make

You will make one HTML file (index.html) that presents your project, and upload it to GitHub. You will submit the GitHub link as your final work.

2 What the File Should Do

The format is free. Using things like graphs, tables, images, and text, you should present your findings in whatever way works best for your project. There is no fixed structure and no required length. Design the page so that it shows your work effectively.

3 The Whole Process at a Glance

  1. Fill in the report template.
  2. Gather your images and any web links.
  3. Open an AI chatbot, upload your filled-in report and your image files.
  4. Paste the prompt from section 7. The chatbot will give you back one HTML file.
  5. Test it in your browser, then upload it to GitHub.

5 Your Report

Use the report template provided with this guide. The template gives you a suggested structure for the PBL module, but you do not have to follow that structure exactly. Change the order, rename sections, add or remove parts so the flow matches your own thinking.

One important rule: follow how images and links are marked. Anywhere you want an image or a live link to appear in your web page, write a marker inside angle brackets:

<image: chart1.png>

<image: https://example.com/photo.jpg>

<https://books.google.com/ngrams/graph?content=media+is&year_start=1800&year_end=2022>

These markers tell the chatbot exactly where each visual goes. You will then upload the actual image files alongside your report when you open the chatbot.

6 Data, Images, and Links

Handling images is the trickiest part of this whole process, so read this section carefully. There is a default way that works for everyone, and there are two other ways available depending on your situation.

Open a normal chat with ChatGPT, Claude, or Gemini. Drag your filled-in report file and all of your image files into the same chat at once. The chatbot will read the angle bracket markers in your report and match each filename to the image you uploaded. Ask the chatbot to embed the images directly inside the HTML as base64 so the final file is one self-contained file with no broken links.

For a Google Books Ngram Viewer link, you have two choices. You can paste the link inside angle brackets in your report, and the chatbot will wrap it in an iframe so the live chart loads inside your page. Or, take a screen capture of the Ngram chart, save it as an image, and treat it like any other image with an angle bracket marker.

If you have a paid ChatGPT or Claude account, you can use Projects. Create a project, upload your report and all your image files into it, and chat inside the project. Everything stays in one workspace so you can keep editing without re-uploading.

If you have a chart or a frequency table from your data that exists only as a screenshot, you can ask the chatbot to read the numbers from the screenshot and redraw it as a clean simple graph or a normal HTML table inside your page. This looks much better than a pasted screenshot.

The chatbot can misread numbers from a blurry image. Check the redrawn version against your original data before submitting.

7 The Prompt

Open the chatbot, upload your filled-in report and all your image files, then paste one of the prompts below. Choose the version that fits how you want your final page to look.

Keep my report as it is. The chatbot will convert your report into a clean HTML page with your visuals in place. Best for students who already wrote the report the way they want it presented.

I have uploaded my project report and my image files. Please turn everything into ONE single self-contained HTML file called index.html that I can upload to GitHub. Follow the flow and structure of my report as it is written, and do not invent content I did not give you. Anything written inside angle brackets in my report (for example <image: chart1.png> or <https://books.google.com/ngrams/...>) is an instruction for you: an image marker tells you to place that image at that point in the page, and a link inside angle brackets tells you to embed it as a live iframe at that point. Embed all images as base64 directly inside the HTML so the page is fully self-contained. Use clean, readable styling with simple colors and good spacing. When you are done, give me a html file named index.html that I can download.

Turn my report into a real web page. The chatbot can change the visual layout, add navigation, hover effects, expandable sections, and other web design elements. It cannot add new content or change your facts.

I have uploaded my project report and my image files. Please turn everything into one interactive HTML web page called index.html that I can upload to GitHub. The content of my report (the text, the data, the meaning) must stay exactly as I wrote it. Do not add new content, do not invent facts, and do not change my findings. However, you have full freedom over the visual presentation: you may reorganize my report into web-style sections, add a top navigation bar that scrolls to each section, use side-by-side layouts, callout boxes, expandable or tabbed sections, hover effects, and any other interactive web elements that make the page more engaging to read. Make it feel like a real web page, not a printed document. Anything written inside angle brackets in my report (for example <image: chart1.png> or https://books.google.com/ngrams/...) is an instruction for you: an image marker tells you to place that image at that point in the page, and a link inside angle brackets tells you to embed it as a live iframe at that point. Embed all images as base64 directly inside the HTML so the page works on its own with no broken links. Use modern, clean styling with good spacing and a readable color scheme. Make the layout responsive so it works on both phone and laptop. When you are done, give me a file named index.html that I can download.

8 Working with the Chatbot Through Iteration

The first version the chatbot gives you will rarely be perfect. The real work is going back and forth with the chatbot to refine the page. Open the result, see what is wrong, and ask the chatbot to fix that specific thing. Keep going until the page is right.

Click each common problem below to see how to handle it:

Image looks too small or blurry

First check the resolution of your original image file. If the original is low quality, replace it with a higher-resolution version. If the original is fine but the chatbot shrunk it, ask the chatbot to display the image at a larger size or at its original resolution.

Image is broken or not showing at all

Tell the chatbot to stop using a local file path or external URL and to re-embed the image as base64 directly inside the HTML.

Google Books Ngram chart not showing

Ask the chatbot to wrap the Ngram link inside an iframe instead of leaving it as plain text or a hyperlink.

Chart redrawn from a screenshot has wrong numbers

Give the chatbot the correct numbers from your original data and ask it to redraw the graph or table.

Layout looks broken or cramped

Ask for a cleaner layout with more spacing. You can also ask for specific colors, fonts, or section orders in plain language.

Chatbot's response is cut off

Ask it to continue the code from where it stopped.

9 Test Your Page Locally

Save the code from the chatbot as a file named index.html on your computer. Double-click the file. It should open in your web browser. Check that all images appear, the text is readable, and any chart or live link looks right. If anything is broken, go back to the chatbot and ask for a fix.
